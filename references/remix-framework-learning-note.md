# Remixでしんどかったのはコードより規約だった

Track To Inventory アプリの開発で Remix を使っていたとき、しんどかったのは JavaScript / TypeScript のコードを書くこと自体ではなく、`routes` の命名規則やファイル配置が挙動に強く関わることだった。

特に、「ロジックは合っているのに、ファイル名や構成のせいで期待どおりに効かない」という場面があり、普通の React / TypeScript で感じる `import / export` の自由さとの違いを強く感じた。ファイル名そのものが設定の一部として働く感覚があり、コードの問題ではなくフレームワーク規約への適応コストで詰まることがあった。

今振り返ると、これは Remix が悪いというより、`file-based routing` と `route module` 規約を前提にしたフレームワーク特有の学習コストだった。便利さと引き換えに、自由度や「普通の React 的な気楽さ」が下がることもある、という理解に変わった。
